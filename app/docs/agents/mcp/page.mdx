# MCP for controlling instances

Every iOS and Android instance Limrun creates exposes its own Model Context Protocol (MCP) server. Three things to know up front:

- **Per-instance, not shared.** There is no single "Limrun MCP endpoint." Each instance has its own URL in `instance.status.mcpUrl`.
- **Auth uses the org API key**, not `instance.status.token`. Pass it as `Authorization: Bearer lim_…`.
- **The tool surface mirrors the SDK's device-control plane**: screenshots, taps, element-tree, text input, app launch/terminate, scrolling, deep links, recording.

This page covers when to reach for MCP, how to grab the URL, and how to wire it into Claude Code, Claude Desktop, Cursor, Codex, and custom MCP clients.

## CLI vs MCP

| Aspect | `lim` CLI | Per-instance MCP server |
|---|---|---|
| **Where it runs** | The agent's shell (Bash / shell tool) | The agent's MCP client transport |
| **Auth** | Org API key in env / config | Same org API key in `Authorization: Bearer` (not `status.token`) |
| **Connection** | Fresh WebSocket per command (or persistent via `lim session start`) | Long-lived MCP session |
| **Tool surface** | Every `lim ios *` / `lim xcode *` / `lim asset *` / `lim android *` subcommand | Device-control subset only (no `xcode build`, no `lim asset push`) |
| **Best for** | Builds, instance lifecycle, anything CLI-shaped | Tight tap/screenshot loops where round-trip latency matters |

**Rule of thumb:** use both. The CLI handles provisioning, builds, and asset management. MCP handles the tight tap-screenshot-elementTree loop, exposed as MCP tools so an agent with native tool-use can call them without shelling out.

## Get the `mcpUrl`

`mcpUrl` is returned in `instance.status` after the instance reaches `ready`. The URL itself does not carry credentials. Auth goes in an `Authorization: Bearer` header, and the bearer is your **org API key** (the same `lim_...` key you authenticate the REST API with), not the per-instance `instance.status.token`.

<CodeGroup>
```ts TypeScript
import Limrun from '@limrun/api';

const lim = new Limrun({ apiKey: process.env['LIM_API_KEY'] });

const instance = await lim.iosInstances.create({
  wait: true,
  reuseIfExists: true,
  metadata: { labels: { name: 'agent-session' } },
});

console.log(instance.status.mcpUrl);
// https://eu-north1-<shard>.limrun.net/v1/ios_<id>/mcp
```

```python Python
from limrun_api import Limrun

client = Limrun()
instance = client.ios_instances.create(wait=True, reuse_if_exists=True)
print(instance.status.mcp_url)
```

```go Go
client := limrun.NewClient()
inst, _ := client.IosInstances.New(ctx, limrun.IosInstanceNewParams{
    Wait:          limrun.Bool(true),
    ReuseIfExists: limrun.Bool(true),
})
fmt.Println(inst.Status.McpURL)
```

```java Java
IosInstance instance = client.iosInstances().create(
    IosInstanceCreateParams.builder()
        .wait(true)
        .reuseIfExists(true)
        .metadata(IosInstanceCreateParams.Metadata.builder()
            .labels(IosInstanceCreateParams.Metadata.Labels.builder()
                .putAdditionalProperty("name", "agent-session")
                .build())
            .build())
        .build()
);
String mcpUrl = instance.status().mcpUrl().orElseThrow();
```

```bash CLI
lim ios get --json | jq -r '.status.mcpUrl'
```
</CodeGroup>

## Wire it into your agent

### Claude Code

The canonical one-liner from the Connect modal:

```bash
claude mcp add simulator --transport http \
  '<mcpUrl>' \
  --header 'Authorization: Bearer lim_<your_api_key>'
```

Replace `<mcpUrl>` with the value from `instance.status.mcpUrl` and `lim_<your_api_key>` with your Limrun API key. Verify with `claude mcp list`.

The console renders the same invocation with your real values prefilled, ready to copy:

### Claude Desktop

Add the URL to your MCP servers config in Claude Desktop's settings:

```json
{
  "mcpServers": {
    "limrun-ios": {
      "type": "http",
      "url": "<mcpUrl>",
      "headers": {
        "Authorization": "Bearer lim_<your_api_key>"
      }
    }
  }
}
```

### Cursor

The console's "Add to Cursor" button on the running instance opens Cursor with the MCP server pre-filled. To configure manually, add a new MCP server in Cursor settings with `<mcpUrl>` and the `Authorization: Bearer lim_<your_api_key>` header.

### Codex Web / custom MCP client

Any MCP client that supports HTTP transport works. Pass the `mcpUrl` as the endpoint and `Authorization: Bearer lim_<your_api_key>` as a header. The Connect modal's **Custom** section gives you those two values directly:

## What the MCP server exposes

The hosted MCP server mirrors the device-control surface of `Ios.createInstanceClient` (iOS) and the equivalent Android client. That includes:

- **Visual**: screenshot
- **State**: element-tree
- **Touch**: tap, tap-element by accessibility selector, scroll
- **Input**: type, press key, toggle keyboard
- **Navigation**: open URL / deep link
- **App lifecycle**: launch app, terminate app
- **Recording**: start / stop video recording

Full reference for each operation, including parameter shapes, lives on the [Run an iOS Simulator](/docs/ios/run-simulator) and [Run an Android Emulator](/docs/android/run-emulator) pages.

## Roll your own MCP server around `lim`

Sometimes the hosted MCP surface isn't quite what you want. Maybe you want to expose `xcode build` as a tool, or wrap multiple `lim` commands into a single higher-level tool. Building your own MCP server is straightforward: wrap the SDK or the CLI in whatever shape you need.

See the [`ios-with-xcode` example](https://github.com/limrun-inc/typescript-sdk/tree/main/examples/ios-with-xcode) for a working pattern that uses `@modelcontextprotocol/sdk` to expose `xcode.xcodebuild()` as a single `build` MCP tool, then mounts it at `http://localhost:3000` for the agent to use.

```ts
import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
import { StreamableHTTPServerTransport } from '@modelcontextprotocol/sdk/server/streamableHttp.js';
import Limrun from '@limrun/api';

const lim = new Limrun({ apiKey: process.env['LIM_API_KEY'] });
const instance = await lim.iosInstances.create({
  wait: true,
  reuseIfExists: true,
  spec: { sandbox: { xcode: { enabled: true } } },
});
const xcode = await lim.xcodeInstances.createClient({
  apiUrl: instance.status.sandbox!.xcode!.url!,
  token: instance.status.token,
});

const mcp = new McpServer({ name: 'xcodebuild', version: '1.0.0' });
mcp.registerTool(
  'build',
  {
    title: 'xcodebuild',
    description: 'Run xcodebuild on the remote Xcode sandbox and install on the simulator.',
  },
  async () => {
    const build = xcode.xcodebuild();
    const chunks: string[] = [];
    build.stdout.on('data', (line) => chunks.push(line.toString()));
    await build;
    return { content: [{ type: 'text', text: chunks.join('\n') }] };
  },
);
```
