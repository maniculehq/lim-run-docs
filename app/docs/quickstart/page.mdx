# Quickstart

Limrun gives a cloud coding agent (Claude Code, Cursor, Codex) hands-on access to a remote iOS
simulator with an attached Xcode sandbox. No local Mac. No `xcodebuild` running on the agent's host.
The agent issues `lim` commands, Limrun handles the build and the simulator, and you watch the device
stream in a browser.

By the end of this guide your cloud coding agent will be building and running an iOS app on a Limrun
simulator.

<Agent>
    ```text
    You are helping me wire a cloud coding agent (Claude Code, Cursor, or Codex) to Limrun so it can build and run an iOS app on a remote simulator.

    Follow the canonical Limrun integration: read the Quickstart at /quickstart.md and the iOS guides at /ios/build-with-xcode.md and /ios/run-simulator.md.

    Concrete steps for me:
    1. Install the CLI with `npm install --global @limrun/cli`. Confirm with `lim --help`.
    2. Authenticate. Either run `lim login` (opens a browser, writes the key to `~/.lim/config.yaml`) or save `LIM_API_KEY=lim_...` to a `.env` file in the project and confirm `.env` is in `.gitignore`.
    3. Install the Limrun skill into the agent's skills directory with `lim skills install`. Pick the right agent and scope when prompted. The bundled skill is named `limrun-ios`.
    4. Create a paired iOS instance and Xcode sandbox with `lim ios create --xcode --reuse-if-exists --label issue=<id> --label repo=<name> --label agent=<name>`. Print the `Signed Stream URL` from the output so I can open it in a browser.
    5. From inside this repo run `lim xcode build .`. Use `--scheme` and `--workspace` if the project has multiple schemes. Each successful build auto-installs and auto-launches the app on the simulator.
    6. After the app launches, verify state with `lim ios element-tree` and drive the UI with `lim ios tap-element --ax-label "<label>"`. Tear down with `lim ios delete` when done.

    Never call local `xcodebuild`. Never spin up a local simulator. Use the `lim` CLI for every build and every device interaction.
    ```
</Agent>

## Prerequisites

- **A Limrun API key.** Sign up at [console.limrun.com](https://console.limrun.com), copy a key
  (prefixed `lim_`), and save it to a `.env` file at the root of your project:

  ```bash
  LIM_API_KEY=lim_...
  ```

  Make sure `.env` is listed in `.gitignore`.

- **Node.js 18 or newer.** The `@limrun/cli` package requires Node 18+. Check with `node --version`.

## Set up the CLI and skill

<Steps>
  <Step>
    ### Install the CLI

    Install `@limrun/cli` globally so the `lim` binary is on your PATH. Your coding agent calls
    this from a shell.

    ```bash
    npm install --global @limrun/cli
    ```

    Confirm the install (the CLI prints its version at the top of the help output):

    ```bash
    lim --help
    ```

    <Callout type="info">
      The `limrun-inc/tap/lim` Homebrew formula is a separate, older Go binary with a partially
      different command surface. These docs are written against `@limrun/cli` — use the npm package
      above, not the brew tap.
    </Callout>
  </Step>

  <Step>
    ### Authenticate

    Pick one of the two paths.

    **Option A: browser login.** `lim login` opens the Limrun console in your default browser,
    spawns a local callback server on port `32412`, and writes the returned key to
    `~/.lim/config.yaml`.

    ```bash
    lim login
    ```

    **Option B: environment variable.** If you already saved `LIM_API_KEY` to `.env` per the
    prerequisites, the CLI picks it up automatically (it loads `.env` on start). You can also
    export it inline:

    ```bash
    export LIM_API_KEY=lim_...
    ```

    The CLI resolves credentials in this order: `--api-key` flag, then `LIM_API_KEY` (including
    from `.env`), then `~/.lim/config.yaml`.
  </Step>

  <Step>
    ### Install the skill into your agent

    `lim skills install` copies the bundled `limrun-ios` skill into the native skills directory
    for your agent (Claude Code, Cursor, or Codex). The skill teaches your agent the full
    `lim ios` and `lim xcode` workflow.

    Interactive (recommended the first time):

    ```bash
    lim skills install
    ```

    The CLI prompts for which agents to target and whether to install at **project** scope
    (current directory) or **global** scope (your home directory).

    Non-interactive, for scripted setup:

    ```bash
    lim skills install --agents claude --scope project
    ```

    Supported `--agents` values: `claude`, `cursor`, `codex`. Repeat the flag to target multiple
    agents in one run. Add `--force` to overwrite an existing skill file.
  </Step>

  <Step>
    ### Create an iOS instance with Xcode attached

    From inside the project the agent is working in, create a paired iOS simulator and Xcode
    sandbox. The `--xcode` flag is what makes `lim xcode build` target this instance. Labels are
    how `--reuse-if-exists` finds the same instance again across agent runs, and how you filter
    listings later.

    ```bash
    lim ios create --xcode \
      --reuse-if-exists \
      --label issue=DEMO \
      --label repo=sample-native-app \
      --label agent=claude
    ```

    The CLI prints something like this when the instance is `ready`:

    ```text
    Created a new iOS instance in 7.2s
    iOS Instance:
      ID: ios_abc123
      Console URL: https://console.limrun.com/stream/ios_abc123
      Signed Stream URL: https://....limrun.net/...?token=...
      Region: us-west
      State: ready
    Xcode Sandbox:
      ID: sandbox_def456
      URL: https://....limrun.net/v1/sandbox_.../xcode
    ```

    Open the **Signed Stream URL** in any browser to watch the live simulator. The signed URL
    embeds its own token, so no API key is needed to view it.
  </Step>

  <Step>
    ### Build and run on the simulator

    Point your agent at a working iOS project. The fastest path is to clone the sample app,
    which has the Limrun skill pre-wired and a stable bundle ID (`com.limrun.sample-native`):

    ```bash
    git clone https://github.com/limrun-inc/sample-native-app.git
    cd sample-native-app
    ```

    Then have your agent run the build:

    ```bash
    lim xcode build .
    ```

    `lim xcode build` syncs the current directory to the remote Xcode sandbox, runs `xcodebuild`
    there, streams logs back over stdout and stderr, and on success auto-installs and auto-launches
    the app on the attached simulator. For multi-scheme projects or workspaces, pass
    `--scheme <name>` and `--workspace <path>`.
  </Step>
</Steps>

## What success looks like

When the steps above land, the agent sees streaming build logs ending in `BUILD SUCCEEDED`, then a
launch confirmation. The browser tab showing the Signed Stream URL updates to the app's first screen.
From here the agent can drive the UI:

```bash
lim ios element-tree
lim ios tap-element --ax-label "Continue"
lim ios screenshot -o /tmp/screen.png
```

When the agent is done, it should clean up the instance:

```bash
lim ios delete
```

Without an ID argument, `lim ios delete` removes the last iOS instance the CLI created for your user.
The pointer is stored in `~/.lim/last-instances.json` and is updated on every `lim ios create`.
