# Run an iOS Simulator

A Limrun iOS instance is a fully booted iPhone, iPad, or Watch simulator running in a cloud pod.
You create it with one REST call, drive it from the SDK over a control WebSocket, and delete it when
the session ends. The control surface is identical whether your code runs on a Mac, a Linux CI box,
a Windows laptop, or inside a coding-agent sandbox.

<Agent>
    You are helping a developer drive a Limrun cloud iOS simulator from code.

    Read the docs in this order:
    1. /ios/run-simulator.md for the instance lifecycle, the Ios.createInstanceClient control surface, and every method below.
    2. /quickstart.md if the developer hasn't installed the CLI or set LIM_API_KEY yet.
    3. /ios/build-with-xcode.md when they need to compile their own app rather than install a prebuilt one.

    The canonical TypeScript flow is:
    1. import { Ios, Limrun } from '@limrun/api'.
    2. Create with await lim.iosInstances.create({ wait: true, reuseIfExists: true, metadata: { labels: { ... } }, spec: { model: 'iphone' } }). wait: true blocks until status.state === 'ready'.
    3. Open the control client with await Ios.createInstanceClient({ apiUrl: instance.status.apiUrl, token: instance.status.token }). Do not use a non-existent iosInstances.createClient.
    4. Drive the simulator. Prefer client.elementTree() and client.tapElement({ AXLabel: '...' }) or { AXUniqueId: '...' } over coordinates. Use AXLabel, not label.
    5. For multi-step interactions use client.performActions([...]). Hardware buttons (home, lock, side, applePay, softwareKeyboard) require paired buttonDown/buttonUp actions.
    6. client.disconnect() and await lim.iosInstances.delete(instance.metadata.id) in finally.

    Never try to run a local iOS simulator or call local xcodebuild.
</Agent>

## Prerequisites

- **A Limrun API key.** Set `LIM_API_KEY` in your environment or `.env` file.
  See [Quickstart](/docs/quickstart) if you haven't yet.
- **An SDK or the CLI.**

  ```bash title="TypeScript SDK"
  npm install @limrun/api
  ```

  ```bash title="lim CLI"
  npm install --global @limrun/cli
  ```

## 1. Provision an iOS instance

The iOS Instance API takes `{ wait, reuseIfExists, metadata, spec }` and returns
`{ metadata, spec, status }` including `apiUrl`, `token`, `signedStreamUrl`, and `mcpUrl`.

```bash title="cURL"
curl -X POST "https://api.limrun.com/v1/ios_instances?wait=true" \
  -H "Authorization: Bearer $LIM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "spec": { "model": "iphone" } }'
```

```ts title="TypeScript"
import { Limrun } from '@limrun/api';

const lim = new Limrun({ apiKey: process.env['LIM_API_KEY'] });

const instance = await lim.iosInstances.create({
  wait: true,
  spec: { model: 'iphone' },
});
```

```bash title="lim CLI"
lim ios create --model iphone
```

With `wait: true` the call blocks until the simulator boots. Cold-start latency is typically a few
seconds; reused instances return almost immediately.

### Configure the spec

| Field | Type | Default | What it does |
| --- | --- | --- | --- |
| `wait` | `boolean` (query) | `false` | Block until the simulator is `ready`. |
| `reuseIfExists` | `boolean` (query) | `false` | Return an existing instance with the same labels instead of creating a new one. |
| `spec.model` | `'iphone' \| 'ipad' \| 'watch'` | `iphone` | Which simulator family to boot. |
| `spec.region` | `string` | scheduler-picked | Pin the pod to a region. |
| `spec.inactivityTimeout` | `string` | `3m` | When to auto-terminate after last control activity. |
| `spec.hardTimeout` | `string` | `0` | Absolute upper bound on instance lifetime. |
| `spec.sandbox.xcode.enabled` | `boolean` | `false` | Provision an embedded Xcode pod next to the simulator. |
| `metadata.labels` | `{ [key]: string }` | unset | Free-form key=value pairs used by `reuseIfExists` and filters. |

### What you get back

| Field | Populated when | What it's for |
| --- | --- | --- |
| `status.state` | always | Lifecycle: `unknown`, `creating`, `assigned`, `ready`, `terminated`. |
| `status.token` | always | Bearer token for every URL below. |
| `status.apiUrl` | `ready` | Base URL for HTTP control. Passed to `Ios.createInstanceClient`. |
| `status.signedStreamUrl` | `ready` | Token-baked stream URL safe to hand to a browser. |
| `status.mcpUrl` | `ready` | Per-instance MCP server. Wire into Claude Desktop, Cursor, or Codex. |

## 2. Connect a device-control client

The control client lives at `Ios.createInstanceClient`. It opens a WebSocket, fetches `deviceInfo`,
and resolves to a typed handle with every device-driving method.

<Callout type="warn">
  The iOS client is a namespace function (`Ios.createInstanceClient`) while the Xcode client is an
  instance method (`lim.xcodeInstances.createClient`). Calling `lim.iosInstances.createClient(...)` 
  will be `undefined`. See [Build with Remote Xcode](/docs/ios/build-with-xcode) for the Xcode variant.
</Callout>

```ts title="TypeScript"
import { Ios, Limrun } from '@limrun/api';

const lim = new Limrun({ apiKey: process.env['LIM_API_KEY'] });

const instance = await lim.iosInstances.create({ wait: true });
if (!instance.status.apiUrl) throw new Error('API URL is missing');

const client = await Ios.createInstanceClient({
  apiUrl: instance.status.apiUrl,
  token: instance.status.token,
});
```

The client caches `deviceInfo` at connect time — `client.deviceInfo.screenWidth`, `screenHeight`,
`udid`, and `model` are available synchronously after the await resolves.

Optional knobs:

```ts
const client = await Ios.createInstanceClient({
  apiUrl: instance.status.apiUrl!,
  token: instance.status.token,
  logLevel: 'info',          // 'none' | 'error' | 'warn' | 'info' | 'debug'
  maxReconnectAttempts: 6,
  reconnectDelay: 1000,      // ms
  maxReconnectDelay: 30000,  // ms
});
```

## 3. Control the iOS instance

With a connected `client`, you have everything you need to drive the simulator from code:

| Capability | What it covers |
| --- | --- |
| Manage apps | Install, launch, terminate, list, and soft-reset apps |
| Read the screen | Element tree, screenshots, server-side video recording |
| Drive input | Tap, type, press keys, scroll, set orientation |
| Batch actions | `performActions([...])` — send a sequence in one call |
| Stream logs | App logs, syslog |
| Low-level shell | `simctl`, `xcrun`, `cp`, `lsof` |

See [Control an iOS Instance](/docs/ios/control-instance) for the full method reference.

## 4. Manage the connection

```ts
client.getConnectionState();
// → 'connecting' | 'connected' | 'disconnected' | 'reconnecting'

const unsubscribe = client.onConnectionStateChange((state) => {
  if (state === 'reconnecting') console.log('reconnecting...');
  if (state === 'disconnected') console.log('dead');
});
unsubscribe(); // when done

client.disconnect(); // closes the WS and stops reconnect attempts
```

For sticky sessions, call `client.keepAlive()` on a cadence to reset the `inactivityTimeout`
counter:

```ts
setInterval(() => client.keepAlive(), 15_000);
```

## 5. Reuse and reap with labels

```ts
// One instance per tenant, reused across reruns.
const instance = await lim.iosInstances.create({
  wait: true,
  reuseIfExists: true,
  metadata: { labels: { tenant: 'acme', purpose: 'preview' } },
  spec: { model: 'iphone' },
});

// Reap every preview instance for a tenant.
const stale = await lim.iosInstances.list({ labelSelector: 'tenant=acme,purpose=preview' });
for await (const i of stale) {
  await lim.iosInstances.delete(i.metadata.id);
}
```

## 6. Tear down

Always disconnect the client and delete the instance when you're done.

```ts title="TypeScript"
try {
  // ... drive the simulator ...
} finally {
  client.disconnect();
  await lim.iosInstances.delete(instance.metadata.id);
}
```

```bash title="lim CLI"
lim ios delete
```

`lim ios delete` without an ID removes the last instance the CLI created in the current directory.

## End-to-end example

```ts title="Full happy path"
import { Ios, Limrun } from '@limrun/api';

const lim = new Limrun({ apiKey: process.env['LIM_API_KEY'] });

const instance = await lim.iosInstances.create({
  wait: true,
  reuseIfExists: true,
  metadata: { labels: { name: 'my-agent-loop' } },
  spec: { model: 'iphone' },
});
if (!instance.status.apiUrl) throw new Error('API URL is missing');

const client = await Ios.createInstanceClient({
  apiUrl: instance.status.apiUrl,
  token: instance.status.token,
});

try {
  const tree = await client.elementTree();
  console.log(`${tree.length} root nodes`);

  await client.openUrl('https://apple.com');
  await client.tapElement({ AXLabel: 'Continue' });
  await client.typeText('hello world', true);

  await client.startRecording({ quality: 7 });
  await client.performActions([
    { type: 'tap', x: 100, y: 200 },
    { type: 'wait', durationMs: 500 },
    { type: 'typeText', text: 'demo', pressEnter: true },
  ]);
  await client.stopRecording({ localPath: 'demo.mp4' });
} finally {
  client.disconnect();
  await lim.iosInstances.delete(instance.metadata.id);
}
```
